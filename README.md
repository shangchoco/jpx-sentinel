# JPX上場廃止銘柄監視システム (JPX Sentinel)

[![AWS](https://img.shields.io/badge/AWS-Fargate-orange)]()
[![Python](https://img.shields.io/badge/Python-3.x-blue)]()
[![Terraform](https://img.shields.io/badge/Terraform-IaC-purple)]()

## 1. プロジェクト概要
東京証券取引所（JPX）の「上場廃止等の公表」情報を自動的に監視し、新規公示を検知した際にSlackへの即時通知とMySQLへのデータ保存を行うシステムです。手動による確認作業を排除し、公示情報の確実なキャッチアップと業務効率化を目的に開発しました。

## 2. システムアーキテクチャ
![System Architecture](img)

* **主要技術:** AWS ECS (Fargate), EventBridge, Python (Selenium), MySQL (RDS)
* **設計思想:** 「イベント駆動型スケジューリング」と「耐障害性」を重視し、保守コストを最小限に抑えたサーバーレス構成を採用しています。

## 3. 主要機能
* **自動スクレイピング:** Seleniumを活用し、JPX公式サイトから非定型データを正規化して抽出。
* **データ整合性の担保:** `INSERT IGNORE`ロジックを実装し、同一銘柄の重複登録を確実に防止。
* **運用可視性の確保:**
    * 処理結果（成功・失敗・対象データなし）をSlackに通知。
    * エラー重要度に応じた動的な通知制御（Error/Critical）を実現。

## 4. 運用処理シーケンス
![Sequence Diagram](img)

* **障害対応戦略:**
    * 一時的なネットワークエラー時は自動リトライを実行。
    * 最終的な失敗時のみ管理者に通知を行い、アラート疲れを防止。

## 5. 技術選定の理由
* **AWS ECS (Fargate):** サーバー運用負荷を低減し、バッチ処理時のみリソースを消費するコスト最適化を実現。
* **Slack Webhook:** チームの日常的なツールと連携し、リアルタイムモニタリング体制を構築。

## 6. テストおよび品質管理
